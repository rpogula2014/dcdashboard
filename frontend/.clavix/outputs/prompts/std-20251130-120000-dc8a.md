Build a DC Dashboard React application following the exact layout specifications in LAYOUT_REFERENCE.md

## Technical Stack
- React 18 with TypeScript
- Ant Design (antd) component library
- React Router for navigation
- CSS for custom styling (row highlights, etc.)

## Implementation Scope

### Phase 1: Core Layout
1. **App Shell** - Main layout wrapper (src/App.tsx)
   - Sidebar (240px expanded / 64px collapsed)
   - Header with title and auto-refresh controls
   - Content area

2. **Sidebar Navigation** (src/components/Layout/Sidebar.tsx)
   - Collapsible with toggle button
   - Navigation groups: Summary, Ship Method, Exceptions, Inventory, Analytics
   - Badge counts on relevant items
   - Red badges for exception counts

3. **Header Component** (src/components/Layout/Header.tsx)
   - Dynamic page title
   - Auto-refresh toggle
   - Last sync timestamp

### Phase 2: Reusable Components
1. **KPICard** (src/components/Dashboard/KPICard.tsx)
   - Title, value, progress bar, footer
   - Color variants: blue, green, orange, red
   - Grid layout: 6 cards per row (responsive)

2. **StatusIcon** (src/components/common/StatusIcon.tsx)
   - Success (checkmark green), Fail (X red), Pending (clock yellow), N/A (dash gray)

3. **OrdersTable** (src/components/Dashboard/OrdersTable.tsx)
   - Exception badge column
   - Sortable columns
   - Row highlighting (critical: red, warning: yellow)
   - Pagination with size selector

### Phase 3: Pages
1. **Summary** - Main dashboard with KPI cards and recent orders
2. **Route Truck** - Orders filtered by Route Truck ship method
3. **Other Ship Methods** - Non-route truck orders
4. **Exceptions** - Orders with exceptions only
